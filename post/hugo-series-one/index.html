Making a static site with Hugo

October 7, 2017 • No Comments #tutorial #goHugo
This article is a part of the hugo-tutorial series: In this series, we will be creating a static Hugo site from scratch, including the theme.

So, Hugo. Why a static site generator, and why exactly Hugo, you ask? Well, a static site is much more faster, secure and simple to host compared to any dynamic site (read: Wordpress). In this part, we’ll set up all the things we need for hugo to work.

Installing Hugo

First, go to Hugo Releases, and scroll down to the downloads section. There, choose the binary that is suitable for your system and download it:

  • For Ubuntu:
    • 32 bit: hugo_0.xx_Linux-32bit.deb
    • 64 bit: hugo_0.xx_Linux-64bit.deb
  • For Windows:
    • 32 bit: hugo_0.xx_Windows-32bit.zip
    • 64 bit: hugo_0.xx_Windows-64bit.zip

Where 0.xx denotes the version number (as of this post, latest version is 0.26, you should grab the latest one).

Now, I’m on Windows, so I’ll be writing instructions for that, but the same steps should apply on other operating systems as well. You should also refer to the Official Hugo Installation Guide

  1. Extract the contents of the .zip file you downloaded and copy them over to C:/hugo/bin
  2. Now rename the executable (eg. hugo_0.26_windows_amd32.exe, it’ll look different depending on what you’re installing it for) to hugo.exe
  3. Add the folder C:/hugo/bin to your path variable, you can find instructions on how to do that at the link above, as well as by googleing “Adding to system path windows (version)”
  4. Restart/ Log off and log back in and check by running hugo version in command prompt, it should show something like this:
> hugo version
Hugo Static Site Generator v0.26 windows/386 BuildDate: 2017-02-27T12:36:30+05:30

And you’re ready to go! (pun intended)

Installing Sublime, PackageControl and Node.js

This portion is not necessary, but installing these will make your life a lot easier in the long run. Oh, and if you are using something else to Autoprefix your css files, skip installing node.js and sublime-autoprefixer

Install Sublime Text and PackageControl

First, go to Sublime Text’s website, grab the latest installer and install it. This should be fairly straightforward.

Install node.js

Go to Node.js, grab the LTS version and install it. Again, the installation should be fairly straightforward. After it’s done, log off/on or restart and then head back to sublime.

Press Ctrl + Shift + P and type install. Package Control: Install Package should be selected. Press enter. Wait till it loads the packages, and then type in Autoprefixer and press enter. Wait for it to install. After that’s done installing, you can use it to autoprefix your .css files.

Autoprefixing will be your best friend when you’re trying to get modern css features like flex-box or even something as simple as transition to run on all browsers. Seeing that a large percentage of people are still using older devices, and the prevalence of browsers like UC (which still doesn’t support flex-box without webkit prefixes), having some backwards compability is never a bad thing.
